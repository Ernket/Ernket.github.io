JIS-CTF Vulnhub TrollCave CTF靶机渗透

前言

这个系列我总共是弄了3台靶机，每完成一个就写一篇，按顺序排依次是简单普通困难，这篇就算是困难难度的了，因为不管是思路还是什么都很考验，有一些点会让你不自觉的在上面耗掉大量的时间，不得不说还是很好玩的，文章流程都是一步到位，我日后会在结尾单独在写一些我失败的操作，走哪哪正确是不存在的除非是你弄的靶机

WEB渗透

用nmap直接是扫到了一个网站，打开看看

小东西长得还挺简单的，上面还写着一个“神他自己都黑不掉这个网站”，这很挑衅，网站的注册功能是关闭的（意料之中），但首页的第二篇文章引起了我的注意

大致意思就是说这个网站的管理员帮别人重置密码重置的多了，觉得很烦，于是就说(这么大的人了不会百度吗)网站添加了新功能，你自己就可以搞定密码重置了，所以我就去google查了一下他这个密码重置的功能，最后在一个网页里面看到了这个功能的路径相关信息

知道了重置密码的路径，那么可以直接尝试着重置一个用户的密码试试

像用户king这种superadmin和用户dave这种admin身份是不能靠这个来重置密码的，所以我们先重置一个普通用户，网站右下角可以看到一些最近加入的普通用户，我们挑一个名字简单一点的来重置

填写了用户名后，上方就会出现一个url，打开这个url，就可以重置密码了

重置完成后，网站默认是登录状态的，先来找找看网站有什么功能

这个时候找到了一个文件上传点，试探性的上传一个php文件，失败

尝试了多种方法后都不行，这里应该是用不了的，所以先放着，这个时候突然想起改密码的url，应该存在着漏洞，改密码那里输入用户名，身份是admin和superadmin的用户都是不允许改密码的，所以这里看看能不能通过修改等于号后面的用户名来绕过，更改密码，所以我们回到起点

在Users那里可以看到所有的用户的名字和身份，所以这里拿dave这个admin来开刀，把原本的onlyme改为dave，果不其然，可以绕过

现在就是以dave这个管理员的身份登录了，这个时候注意到了网站的内容变化了，多了一些原本“看不见”的文章，翻一翻，看到了一个关于网站新功能介绍的文章

翻译过来是这样的

当coderguy在上个月的服务器崩溃之后，不得不重新创建一个网站的时候（记得备份，孩子），他在其中添加了一个很酷的新功能，这个新功能被称为博客放行级别，如果你看到了新的文章，你会在其下方看到一个下拉列表，那可以让你设置哪些用户可以看到博客，有以下几种级别：
Level 0: 任何人都可以看到，所以权限为访客的用户也能看到你的文章
Level 1: 仅成员可见，因此只有登录的用户才能看见他(就像这篇文章！)
Level 2: 仅经常访问的成员，或者说是正式会员才可见，所以只有比较酷的孩子才能看到
Level 3-5: 用于以下身份的用户 mods,admins和superadmins，因此我们可以更好的讨论网站问题或者向King写下情书

我们的目标应该就是往最高权限前进，也就是用户king，看完隐藏的文章后，dave用户的inbox并没有东西，所以我们继续切换管理员用户来看看有没有意外的惊喜

用户coderguy空的，继续换

我们切换到用户dragon的时候，发现inbox有信件了，是King发送的一封名为password的信件，点开来看看

尽量少用它
打死都不要分享出去
PS:我很严肃的说

尴尬的是，我一开始看到密码的时候还以为他是给加密的，去查了一下发现不对劲，谁没事干会发密文….然后直接用这串密码登录King，登录成功

现在我是superadmin了，根据上面的博客等级，我们的等级应该是最高的了，来看看有没有新的隐藏内容，回到home里面，发现有一篇名为sudo give me sudo的文章有点东西，点开来

如果想让我在这个网站上做更多的工作的话，那你就真的得给我一些权限了，我知道这不是一个好的点子，我可以交互式的使用rails这个用户，也许我们可以把它区分开来（区分开来可以理解为用户权限低，干涉不了系统）

这个时候我们回到上方选项，发现多了一个功能，Admin panel，管理控制面板

才发现原来不是什么过滤防护，而是上传文件的功能关闭了，可以看到还有注册功能也给关闭了Enable registration，这里把文件上传先打开（注册功能随你喜好开），接着回到文件上传处，发现文件可以上传了，但是要提醒的一点是，这个网站的编程语言是用ruby写的，所以php、asp之类的木马是回弹不了shell的，它根本就不解析

这个时候看到上传处下方有一个Alternate file name，后来在seebug上面查到了CVE-2012-4522，Ruby本地文件创建漏洞，类别为越权访问，所以这个Alternate file name应该是存在目录遍历的，利用这一点，我们可以直接将我们攻击机的公钥存放到对方机器，做一个免密码ssh登录

在翻找敏感文件的时候，碰巧在king用户目录下找到了一个js文件，内容如下

var http = require("http");
var url = require("url");
var sys = require('sys');
var exec = require('child_process').exec;

// Start server
function start(route)
{
    function onRequest(request, response)
    {
        var theurl = url.parse(request.url);
        var pathname = theurl.pathname;
        var query = theurl.query; 
        console.log("Request for " + pathname + query + " received.");
        route(pathname, request, query, response);
    }

http.createServer(onRequest).listen(8888, '127.0.0.1');
console.log("Server started");
}

// Route request
function route(pathname, request, query, response)
{
    console.log("About to route request for " + pathname);
    switch (pathname)
    {
        // security risk
        /*case "/ping":
            pingit(pathname, request, query, response);
            break;    */

        case "/":
            home(pathname, request, query, response);
            break;    

        case "/calc":
            calc(pathname, request, query, response);
            break;    

        default:
            console.log("404");
            display_404(pathname, request, response);
            break;
    }
}

function home(pathname, request, query, response)
{
    response.end("<h1>The King's Calculator</h1>" +
            "<p>Enter your calculation below:</p>" +
            "<form action='/calc' method='get'>" +
                "<input type='text' name='sum' value='1+1'>" +
                "<input type='submit' value='Calculate!'>" +
            "</form>" +
            "<hr style='margin-top:50%'>" +
            "<small><i>Powered by node.js</i></small>"
            );
}

function calc(pathname, request, query, response)
{
    sum = query.split('=')[1];
    console.log(sum)
    response.writeHead(200, {"Content-Type": "text/plain"});

    response.end(eval(sum).toString());
}

function ping(pathname, request, query, response)
{
    ip = query.split('=')[1];
    console.log(ip)
    response.writeHead(200, {"Content-Type": "text/plain"});

    exec("ping -c4 " + ip, function(err, stdout, stderr) {
        response.end(stdout);
    });
}

function display_404(pathname, request, response)
{
    response.write("<h1>404 Not Found</h1>");
    response.end("I don't have that page, sorry!");
}

// Start the server and route the requests
start(route);

代码第18行的解释：

第一行ver http = require("http")，表示加载http模块，然后，第18行调用了http模块的createServer的方法，创建了一个服务器实例，第9行的request和是一个对象，表示客户端的http请求，response也是一个对象，表示服务器端的http回应。最后的listen表示启动服务器实例，监听本机的8888端口。

接着看到48-68行，可以看到一个参数name='sum' value='1+1',然后67行有一个eval(sum)，没有过滤任何字符，所以这一块只可以进行注入的，先来测试一下正常的结果
